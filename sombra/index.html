Assistente Virtual Diagnóstika

Modo Sombra · observa demandas reais do MegaZap · nada é enviado ao cliente
🟢 Como funciona: cada cartão é uma demanda real capturada do WhatsApp (via n8n). O assistente sugere a classificação e a resposta. Clique ✅ Usaria, ⚠️ Ajustaria ou ❌ Errada. O placar acima mede o quanto o assistente já está "afiado". Quando ART passar de ~90% ✅, migramos esse fluxo pro atendimento real.
🟢 Captura ao vivo via Evolution. Cada cartão agora puxa o histórico real do WhatsApp em tempo real (Evolution API → n8n → Supabase Realtime). Quando o cliente manda mensagem nova, ela aparece automaticamente no cartão e a sugestão da IA pode ser revisada. Confirmado: a API do MegaZap não tem evento de "mensagem recebida" (só atendimento), por isso a captura é via Evolution.
📋 Pendências dos últimos 10 dias (07/06 → 17/06/2026). Cada caso traz o avaliador de aprovação: a IA classifica a demanda, confere a documentação obrigatória (padrão NBR 16280) e dá o veredito automático — apto a emitir a Autorização de Reforma (ARO) ou pendente. Marque os documentos recebidos e o veredito recalcula. Vermelho = aguarda Diagnóstika · Âmbar = aguarda cliente. Nada é enviado — é treino/triagem.
📐 Padrão de aprovação de reforma (Diagnóstika · NBR 16280:2020). Para liberar uma obra exigimos: 1) ART/RRT do responsável técnico vigente · 2) Memorial descritivo / plano de reforma assinado · 3) Termo de responsabilidade civil (proprietário + prestador) · 4) Identificação completa (nome, CPF, bloco/apto, prestador). Com os 4 OK e sem intervenção estrutural pendente de laudo → a Diagnóstika emite a ARO (Autorização de Reforma). É esse fluxo fixo que vamos automatizar.
